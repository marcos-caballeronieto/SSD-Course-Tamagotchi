A high-fidelity virtual pet application built to showcase the power of **Spec-Driven Development (SDD)**. This project demonstrates how autonomous AI agents can build complex, state-heavy interactive systems by following a rigorous, documentation-first architectural approach.

---

## 🐣 Project Overview

**Tiny Tamagotchi** is a living digital ecosystem. Unlike simple "vibe-coded" projects, every line of code here was generated by an AI agent operating under strict constraints, following a predefined roadmap, and self-validating through automated testing.

### 🎮 Game Features
- **Real-Time Vitals**: Hunger, Happiness, Health, and Energy that decay even when you are away.
- **Dynamic Evolution**: Watch your pet grow from an **Egg** to a **Senior** ghost.
- **Advanced Health System**: Neglect leads to sickness; recovery requires active care.
- **Interactive Reactions**: Unique animations and "Easter egg" reactions for every interaction.
- **State Persistence**: Your pet's state is saved locally, surviving browser refreshes.
- **Dev Tools**: Integrated debug menu for rapid testing of evolution stages and stat manipulation.

---

## 🧠 What is Spec-Driven Development (SDD)?

This repository is a **living specification**. The core philosophy is that **the documentation is the source of truth**, and the code is merely a derivative of that truth.

### The SDD Cycle
1.  **Mission First**: Define the "Why" in `specs/mission.md`.
2.  **Strict Guardrails**: Define the "How" and "What" in `specs/techstack.md` and `agents.md`.
3.  **Feature Scoping**: Every feature starts as a `requirements.md` and `feature_plan.md`.
4.  **Phased Execution**: Implementation follows a deterministic `ROADMAP.md`.
5.  **Continuous Verification**: Automated Playwright tests ensure the code matches the specs.

> [!IMPORTANT]
> **Why this matters:** SDD prevents "context decay" in AI models. By keeping small, focused specification files, the AI never loses track of the project's architecture or goals, leading to zero-bug handoffs and modular growth.

## 🏗️ Repository Architecture

The project is structured to be "Agent-Readable":

```text
├── 🤖 specs/                 # The Source of Truth
│   ├── agents.md             # The Core Protocols (Agent Constitution)
│   ├── mission.md            # Project Vision & Constraints
│   ├── techstack.md          # Technology & Performance Standards
│   └── features/             # Atomic Feature Documentation
├── 🗺️ ROADMAP.md             # Real-time state of development
├── 🛠️ skills/                # Domain-specific instructions for the AI
└── 💻 app/                   # The implementation (React + Vite)
```

## 🛠️ Technical Stack

- **Framework**: React 19 + Vite
- **Logic**: TypeScript (Strict Mode)
- **State**: Zustand (Atomic State Management)
- **Styling**: Vanilla CSS + CSS Modules (Custom HSL System)
- **Animations**: Framer Motion
- **Testing**: Vitest & Playwright

## 🚀 Getting Started

### Prerequisites
- Node.js (Late LTS)
- npm or pnpm

### Installation
1.  **Clone the Repo**:
    ```bash
    git clone https://github.com/marcos-caballeronieto/SSD-Course-Tamagotchi.git
    cd Tamagotchi
    ```
2.  **Install & Start**:
    ```bash
    cd app
    npm install
    npm run dev
    ```
